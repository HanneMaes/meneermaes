In deze opdracht onderzoek je hoe een **processor (CPU)** werkt en hoe verschillende onderdelen samenwerken om programma’s uit te voeren.

Je leert niet alleen wat een processor doet, maar ook hoe je doelgericht informatie opzoekt, beoordeelt en verwerkt tot een duidelijk digitaal eindproduct.

# Doelen

- Je kan de werking en opbouw van een processor uitleggen met de **correcte vaktaal**.
- Je kan informatie doelgericht opzoeken in **meerdere bronnen**.
- Je kan bronnen **kritisch beoordelen** op betrouwbaarheid en bruikbaarheid.
- Je kan informatie verwerken tot een **gestructureerd** digitaal document.
- Je kan **reflecteren** over je eigen leerproces en gebruikte strategieën.

# Begrippen

De volgende begrippen moeten correct gebruikt worden:
- CPU (Central Processing Unit)
- ALU (Arithmetic Logic Unit)
- Control Unit
- Registers
- Cache (L1, L2, L3)
- Klokfrequentie
- Fetch–Decode–Execute-cyclus
- Instruction set
- Core / multicore
- Von Neumann-architectuur

# Eindproduct

Je eindproduct is een digitaal document, gemaakt in een tool naar keuze.

Vereisten:
- Duidelijke structuur (titels, tussentitels)
- Correcte vaktaal
- Schema's of afbeeldingen (met bronvermelding)
- Eigen formuleringen (niet kopiëren)
- Bronnenlijst onderaan

## 1. Zoekstrategie

### Zoektermen & bronnen

Noteer kort maar duidelijk:
- Welke zoektermen heb je gebruikt?
- Welke bronnen heb je geraadpleegd?
  - Websites
  - Video’s
  - ...
- Waarom heb je voor deze bronnen gekozen?

### Criteria

Gebruik minstens **3 verschillende bronnen** en beoordeel ze aan de hand van deze criteria:
- Betrouwbaarheid
- Actualiteit
- Doelgroep
- Objectiviteit

## 2. Vragen

Beantwoord onderstaande vragen op basis van je onderzoek:

### Deel 1: Wat is een processor?

1. Wat is een processor en wat is zijn functie in een computer?
2. Waar bevindt de processor zich in een computersysteem?

### Deel 2: Von Neumann-architectuur

1. Wat wordt bedoeld met de von Neumann-architectuur?
2. Waarom werd deze architectuur ontwikkeld en in welke historische context?
3. Hoe communiceren CPU, geheugen, invoer en uitvoer met elkaar volgens de von Neumann-architectuur?
4. Waarom worden programma-instructies en data in hetzelfde geheugen opgeslagen?
5. Wat is het von Neumann-knelpunt?
6. Wat is het verschil tussen de von Neumann-architectuur en de Harvard-architectuur?

### Deel 3: Interne opbouw van de processor

1. Uit welke onderdelen bestaat een processor?
2. Wat is de functie van:
- de ALU?
- de Control Unit?
- registers?
- cachegeheugen?

### Deel 4: Werking van de processor

1. Leg stap voor stap de fetch–decode–execute-cyclus uit.
2. Waarom is klokfrequentie belangrijk?
3. Wat is het verschil tussen een single-core en multi-core processor?

### Deel 5: Processor in de praktijk

1. Waarom zijn moderne processors efficiënter dan oudere?
2. Welke invloed heeft de processor op prestaties bij:
- Gamen
- Videobewerking
- Dagelijks computergebruik?

## 3. Reflectie

Beantwoord op het einde deze vragen:
1. Wat vond je **moeilijk** aan deze opdracht?
2. Welke strategie hielp je om de leerstof te **begrijpen**?
3. Wat zou je volgende keer **anders aanpakken** bij een opzoekopdracht?
